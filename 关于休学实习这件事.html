关于休学实习这件事

又到了白色相簿的季节。。反正学校也回不去，不如先实习，一年后再看。投简历只用了BOSS和拉钩。BOSS直拒需要先简单介绍自己，hr 感兴趣才会收简历。拉钩直接投，但聊天很少有回，而且一般两重态。

终于在第二周，陆续收到了面试邀请。

1st

第一次面试来自拉钩，某海淘电商实习岗。首先是笔试，题目偏重Java基础和算法，简单记录下：

  • 比较Integer
// 似乎能推理得出
Integer a = 10;
Integer b = 10;
Integer c = Integer.valueOf(10);
Integer d = new Integer(10);

Integer e = 127;
Integer f = 127;
Integer g = 128;
Integer h = 128;

System.out.println(a==b); // true
System.out.println(a==c); // true
System.out.println(a==d); // false
System.out.println(c==d); // false

System.out.println(a.equals(b)); // true
System.out.println(c.equals(d)); // true
System.out.println(a.equals(c)); // true

// Integer为-128到127之间的所有整数对象提供缓存，需要的时候直接return，故引用相同。
System.out.println(e == f); // true
// 可以猜出来，源码中，超出某个范围（-128~127）只能重新new一个，故引用不同。
System.out.println(g == h); // false
  • 至少两种方式实现 Singleton
  1. Enum
// 首选，简单高效，缺点是不能 lazy loading
public enum Singleton {
    INSTANCE;
    public void foo() {}
}
  1. SingletonHolder:
// 如需 lazy loading，推荐此方法
public class Singleton {  
    // 私有静态内部类
    private static class SingletonHolder {  
        // 内存中只存在一个，只能赋值一次
        private static final Singleton INSTANCE = new Singleton();  
    }  
    // 私有构造函数，防止此类被new出来
    private Singleton() {}  
    public static final Singleton getInstance() {  
        return SingletonHolder.INSTANCE; 
    }  
}

……

笔试后等待面试，自觉没戏。前面有些拿不准，最后一题实现 CurrentHashMap 无从下手，只写了思路。话说，平时太依赖 IDE 自动提示，拿起笔有种没写过代码的感觉-_-||

嘛，最后6进2，第一次面试完结（并没有面试？）。

2nd

好在笔试刚结束就有另一家公司打来电话，是一家编程学习平台。之前在 BOSS 聊过，电话里也说明了学历问题，感觉有戏。去之前有三轮线上笔试，在他们开发的在线 IDE 中完成三个 Web 小项目（其中一个。他们的 IDE 没有代码提示，而且每次粘贴不能超过五行，体验欠佳。

以下破解过程：

先尝试用少量数据分析 API，写 Python 脚本模拟登陆并提交代码，太麻烦放弃。不过发现了后端好像没有数据长度限制。改变思路从 js 入手，通过 console 给粘贴事件绑定别的操作以覆盖原方法。试过几种方案，绑是能绑上，但无法覆盖。

最后解决是直接定位 js 文件，找到事件绑定的语句，删掉换成 console.log(0) 用于测试，最后用猴油重载整个 js 文件，完美。

PS. 后来才知道他们产品的优势是教师可以实时观看学生的编码过程…

到场先是写项目。两个人，从10点写到14点，没顾上午饭。题目是某小游戏后端，当时觉得要求过于简单，因为是在线IDE，就用 Spring boot + H2 内存模式简单搭了 CRUD，又写了套 API 配合前端，演示用 Postman 。

面试先自己介绍刚写的项目，面试官时不时提些问题，对咱这种记忆力不好的家伙来说，体验还行。但面试官说，他们想考察的是算法方面，我用 H2，完全看不出来…嘛，技不如人，再次被拒。

3rd

第三家去的比较早，直接和 CTO 聊了起来（尬聊，maybe）。关于项目，手写快排，Spring 原理，sql 注入，修电脑等方面，历时一个半小时。CTO 是阿里出来的架构师，环境也不错。只是临走前说，本想招应届生，不确定我能否留在公司，下周给我答复。

可以理解为委婉的拒绝，而当时 naive，还傻等了一周。

4th

为了省钱住在市郊。代价就是公交转了三次，说起来西湖的电公交，坐久了先是四肢发麻，接着是嘴唇，随后就意识模糊。刚下车没啥感觉，一秒后我的胃，是有生之年系列呢！

虽说是第一次面大厂，阿里系的。但本身就抱着打酱油的心态，冲淡了紧张感。

好吧，嘴上说着不紧张，但还是害怕的不得了。进了大门，假装镇定，心中默念，我是来打酱油的我是来打酱油的。。径直走向前台小姐姐。

前台小姐姐：先生您好，请问您是来面试的吗？

我：是的。

前台小姐姐：那您是来面外包岗还是内部岗的？

？？？是不是少了个实习岗？话说实习岗应该算内部岗吧？要不是那就尴尬了，但来不及纠结，于是乎

我：我是来打酱油的。

前台小姐姐：。。。

我：。。。

我（反应过来：不对，我是来面实习岗的。

前台小姐姐：那你找个地方把这张表填一下吧。

绝对被嫌弃了啊！为什么会这样啊！！所谓表格是一张入职申请表，又是第一次~上面的内容感觉跟简历差不多，看周围的求职者很认真的写，想了想，还是只填了基本信息，问了前台，说不写也行，随后打电话叫 hr 下来领人。

技术面是组里的 leader 负责，简历上写的挨个问，再根据回答逐渐深入。虽说答得不是很好，周一还是接到了复试通知。

莫非时来运转？

复试由 hr 部门主管负责。先做个自我介绍（面试这几家第一次自我介绍~>_<~+），接着问了下前面提到的经历。

Q: 有没有数据处理方面的经历？

A: 大一的时候参加过数学建模大赛，当时的题目是 O2O 租车平台资源的最优分配，拿过省奖。

Q: 能具体说下细节吗？

A: …当时用 C 语言实现的动态规划（心虚，时隔两年没看

Q: 具体呢？

A: 背包算法

Q: 心累暂时不写了。刚刚等到了被拒通知（化悲愤为食欲

苦果早在六年前种下，一切只不过是咎由自取。

总结一下，学历是个大问题（终究是能力不够强）。

每次都能听到：我们这个岗位是面向18届的，你的情况比较特殊，如果有结果我们会通知你的。

或者：专科不行呢，抱歉。

  • 如果你还年轻小，别急，哪怕是复读，以后的路会好走很多。

  • 如果你是普通本科，没有亮眼的简历 & 能力，别出来作死。

  • 如果你是专科，不擅长社交，建议专升本。

  • 如果你和我一样，欢迎和我联系，或许以后的我能给你一些建议。

  • 当然，如果你有钱，当我没说。// 谜之声：抱歉，有钱真的是可以为所欲为~

还有，简历一定要认真对待！！很重要！！本人写的 V1 版体无完肤，V2 版无人问津，V3 版，只是增加了技术栈的具体描述，优化了一下细节。投了两家都给了面试邀请，仿佛在做梦（白日梦罢了。
